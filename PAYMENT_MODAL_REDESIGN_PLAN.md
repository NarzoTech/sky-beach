# Payment Modal Redesign Plan

## Current Issues

### 1. Single Modal for All Order Types
- Same payment modal used for Dine-in, Take-away, and Delivery
- Hidden/shown fields based on order type creates confusion
- Too many hidden inputs and conditional displays

### 2. Complex Running Order Payment
- Separate modal for running orders
- Duplicate code between checkout and running order modals
- Discount calculation in two places

### 3. UX Problems
- Payment modal is cluttered with too many sections
- Quick amount buttons are small and hard to use
- Order summary takes too much space
- Payment methods section is confusing

---

## Redesign Goals

1. **Clean, focused interface** for each order type
2. **Unified payment component** reusable across all flows
3. **Touch-friendly** quick payment buttons
4. **Clear visual hierarchy** - total prominently displayed
5. **Streamlined flow** - fewer clicks to complete payment

## Helper Function: Quick Amount Suggestions

```php
// app/Helpers/pos_helpers.php

function suggestQuickAmounts($total) {
    $total = ceil($total);
    $suggestions = [];

    // Round up to nearest 100
    $nearest100 = ceil($total / 100) * 100;
    if ($nearest100 > $total) {
        $suggestions[] = $nearest100;
    }

    // Round up to nearest 500
    $nearest500 = ceil($total / 500) * 500;
    if ($nearest500 > $total && !in_array($nearest500, $suggestions)) {
        $suggestions[] = $nearest500;
    }

    // Round up to nearest 1000
    $nearest1000 = ceil($total / 1000) * 1000;
    if ($nearest1000 > $total && !in_array($nearest1000, $suggestions)) {
        $suggestions[] = $nearest1000;
    }

    // Add +500 and +1000 options
    $plus500 = $nearest100 + 500;
    $plus1000 = $nearest100 + 1000;

    if (!in_array($plus500, $suggestions)) {
        $suggestions[] = $plus500;
    }
    if (!in_array($plus1000, $suggestions)) {
        $suggestions[] = $plus1000;
    }

    // Sort and take top 5
    sort($suggestions);
    return array_slice($suggestions, 0, 5);
}
```
